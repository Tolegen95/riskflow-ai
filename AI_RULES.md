# AI_RULES

- Всегда сначала читать `PROJECT_CONTEXT.md`.
- Не сканировать весь проект без необходимости.
- Открывать только нужные файлы и небольшие релевантные фрагменты.
- Не переписывать код целиком без причины.
- Следовать существующей архитектуре: монолитный Flask `app.py`, SQLite, Jinja templates.
- При изменении архитектуры, структуры БД, запуска или ключевой бизнес-логики обновлять `PROJECT_CONTEXT.md`.
- Перед правками проверять текущий `git status` и не откатывать чужие изменения.
- Если данных не хватает, делать разумные предположения и явно помечать их как `предположение`.
